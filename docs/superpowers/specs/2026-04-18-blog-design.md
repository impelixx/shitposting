# Блог — Дизайн и архитектура

## Цель

Личный блог где автор пишет обо всём что приходит в голову (технологии, личное, книги и т.д.), а читатели могут комментировать статьи.

---

## Дизайн

### Цветовая схема — Sunset

| Элемент | Цвет |
|---|---|
| Шапка | `#1c1917` (почти чёрный) |
| Фон страницы | `#fafaf9` (тёплый белый) |
| Акцент | `#f97316` (оранжевый) |
| Теги | `#ffedd5` фон / `#c2410c` текст |
| Текст основной | `#1c1917` |
| Текст второстепенный | `#78716c` |
| Разделители | `#e7e5e4` |
| Сайдбар фон | `#f5f5f4` |

### Типографика

- Заголовки и текст статей: **Georgia, serif**
- UI-элементы (навигация, теги, мета): **system-ui, sans-serif**

### Компоненты

**Шапка (nav):**
- Логотип + подзаголовок «Пишу обо всём что приходит в голову» слева
- Ссылки (Статьи / Теги / О блоге) справа
- Поисковая строка под логотипом (только на главной)

**Дата-значок:**
- Оранжевый прямоугольник с скруглёнными углами
- Число крупно (жирный), месяц мелко под ним
- Располагается слева от заголовка статьи

**Теги:**
- Пилюли с оранжевым фоном `#ffedd5` и тёмно-оранжевым текстом
- Располагаются над заголовком статьи

## Страницы

### Главная (`/`)

Двухколоночный layout:

- **Лента** (основная колонка): статьи в хронологическом порядке, каждая содержит:
  - Теги
  - Дата-значок + заголовок + отрывок
  - Метаданные: время чтения, счётчик комментариев
- **Сайдбар** (220px): облако тегов

### Статья (`/posts/[slug]`)

- Шапка сайта без поиска, ссылка «← Все статьи» вместо «Статьи»
- Центрированная колонка, max-width 640px
- Структура: теги → дата-значок + заголовок + метаданные → разделитель → текст
- Под текстом: секция комментариев с полем для ввода

### Теги (`/tags/[tag]`)

- Та же лента что на главной, отфильтрованная по тегу
- Заголовок страницы: `#[tag]`

### О блоге (`/about`)

- Центрированная колонка, свободный текст

### Вход (`/login`)

- Форма: логин + пароль
- При успехе → JWT в httpOnly cookie → редирект на `/admin`

### Админка (`/admin`)

- Защищена JWT, доступна только автору
- Список всех статей (включая черновики) с кнопками редактировать/удалить
- Markdown-редактор для написания и редактирования статей
- Поля: заголовок, slug, теги, текст (markdown), черновик/опубликовать
- Загрузка картинок в редакторе → `/api/upload` → Cloudflare R2

## Архитектура

### Стек

| Слой | Технология |
|---|---|
| Фронтенд | Next.js 14, TypeScript |
| Бэкенд | Go (Gin или Chi) |
| База данных | PostgreSQL |
| Хранилище картинок | Cloudflare R2 |
| Поиск | Meilisearch |

### Схема взаимодействия

```
Next.js (TS)
    ↕ REST API
Go API
    ↕ PostgreSQL   — статьи, теги, комментарии
    ↕ Cloudflare R2 — картинки в статьях
    ↕ Meilisearch   — полнотекстовый поиск по статьям
```

### Модели данных

**Post**
```
id         UUID
title      TEXT
slug       TEXT (unique)
body       TEXT (markdown)
excerpt    TEXT
tags       TEXT[]
published  BOOL
created_at TIMESTAMP
updated_at TIMESTAMP
```

**Comment**
```
id         UUID
post_id    UUID (→ Post)
author     TEXT
body       TEXT
created_at TIMESTAMP
```

**Tag**
```
slug  TEXT (unique)
label TEXT
```

### API эндпоинты (Go)

| Метод | Путь | Описание |
|---|---|---|
| GET | `/api/posts` | Список статей (пагинация, фильтр по тегу) |
| GET | `/api/posts/:slug` | Одна статья |
| POST | `/api/posts` | Создать статью (auth) |
| PUT | `/api/posts/:slug` | Обновить статью (auth) |
| DELETE | `/api/posts/:slug` | Удалить статью (auth) |
| GET | `/api/posts/:slug/comments` | Комментарии к статье |
| POST | `/api/posts/:slug/comments` | Добавить комментарий |
| GET | `/api/tags` | Все теги |
| GET | `/api/search?q=` | Поиск через Meilisearch |
| POST | `/api/upload` | Загрузить картинку → R2 (auth) |

### Авторизация

- Только один пользователь (автор). Логин/пароль → JWT токен.
- Защищённые эндпоинты: создание/редактирование/удаление статей, загрузка картинок.
- Комментарии — без авторизации, только имя и текст.

### Поиск

- При публикации/обновлении статьи Go-сервис индексирует её в Meilisearch (title + excerpt + body).
- Поисковая строка на главной делает запрос к `/api/search?q=...`.
- Meilisearch поддерживает русский язык и опечатки из коробки.

## Что не входит в scope

- Подписка на рассылку (newsletter)
- Лайки / реакции на статьи
- Система регистрации читателей
- Тёмная тема
- Мобильное приложение
